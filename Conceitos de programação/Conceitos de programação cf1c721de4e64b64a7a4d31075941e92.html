Conceitos de programação

Aula 1 Pensamento computacional

Pilar: Decomposição

Dado um problema complexo, devemos decompor o problema em problemas menores, de modo que fique mais fácil resolvê-los.

A ordem de resolução dos problemas menores pode ser em sequência ou paralela, dependendo se o problema é dependente entre eles ou não.

Praticar maneiras distintas de decompor o mesmo problema.

Pilar: Padrões

Um dos motivos para determinar padrões é generalizar uma solução para diferentes problemas, similares, de modo a ter resoluções padronizadas e mais ágeis para serem utilizadas.

Aplicações + comuns:

  • Classificação de dados
  • Reconhecimento de imagens
  • Reconhecimento de fala
  • Análise de cenas
  • Classificação de documentos

Pilar: Abstração

Abstrair/Generalizar é tornar algo mais amplo, geral, determinar um conjunto de elementos ou objetos numa classe mais ampla.

A forma para classificar esses dados é por meio dos pontos essenciais e características. De forma que o que é item de detalhamento fica de fora da análise.

Pilar: Algoritmos

O processo de resolução de problemas usando algoritmos é chamado de step by step. É o processo de instruções que diz o que fazer, como fazer e quando fazer.

Estudo de caso conceitual - Perdido na floresta

Sobrevivência

  • Identificar mecanismos
  • Recursos comuns
  • Detalhes mais importantes

Abstraindo o problema maior - sobrevivência - chegamos em padrões, como o fogo, que se repete em mais de uma solução.

Podemos aplicar um algoritmo em algumas das etapas de sobrevivência para criar um fluxo de tarefas a serem seguidas para a resolução do problema.

Como aprimorar a habilidade de pensamento computacional?

Permitindo que o aluno explique suas decisões e seu processo de desenvolvimento.

Tipologia e Variáveis

Tipos de dados

  • Numéricos
    • Inteiros - positivos ou negativos
    • Reais - positivos ou negativos com casas decimais
  • Caracteres
    • Qualquer dado fora da classe numérica
  • Lógico
    • Booleano

Variável - É um tipo de estrutura mutável, que varia dentro do seu valor; Ela é inconstante; Pode ser sobrescrito; Mutável.

Pode receber qualquer valor de um determinado conjunto de valores, mas não pode diferir do tipo definido. Ex: Duas variáveis do tipo string não podem gerar uma variável do tipo numérico.

Regras para definição de nome de variável

  1. Atribuição de um mais caracteres
  1. Primeira letra - não número
  1. Sem espaços em branco
  1. Não pode utilizar palavras reservadas da linguagem utilizada. Ex.: Em Javascript, voce não pode definir uma variável com a palavra let pois ela é reservada para a linguagem definir variáveis.

Uma constante é um tipo de variável inalterável.

Estruturas de repetição

Laços, controle de fluxo, malhas de repetição e loop, são termos ligados a estruturas de repetição.

Condições de parado são definidas para evitar um loop infinito da estrutura de repetição.

Utilizar a estrutura de repetição acarreta na redução de linhas, facilidade em dar manutenção no código, redução de erros.

Exemplos:

Enquanto ...faça

Repita…até

Pare… De … até …faça

Há possibilidade de usar uma estrutura condicional dentro de uma estrutura de repetição (e vice versa)

Vetores e matrizes

Vetor

Definição formal: “Um vetor é caracterizado por uma variável dimensionada com tamanho pré-fixado.”

Vetor é uma sequência de (tipo de) dados.

Matriz

Definição formal: “Matriz é uma tabela organizada em linhas e colunas no formato m x n, onde m representa o número de linhas (horizontal) e n o número de colunas (vertical).”

Funções

Similar ao conceito de função matemática.

A partir de um elemento A que é recebido na função, ela dará a saída de um elemento B.

Definição formal: As funções, ou sub-rotinas, são blocos de instruções que realizam tarefas específicas.

Vantagens:

  • Modularização do problema
  • Código mais claro e conciso
  • Reutilização de instruções

Instruções de Entrada e Saída

Definição formal: Consiste na impressão dos dados do mundo abstrato, digital por meio de ação de alguma interface.

Os formatos podem variar desde simples arquivos binários até complexas querys de banco de dados.

Existem dois tipos de saídas dentro de um programa:

  • Saída programada
    • Condicional
      • Normalmente aguarda o dispositivo disparar a condição
    • Incondicional
  • Saída por interrupção
    • Geralmente definida pelos periféricos

Aula 4 Linguagens de Programação

O que é uma linguagem de programação ?

Método padronizado composto por um conjunto de regras sintáticas e semânticas de implementação de um código fonte. [Conjunto de palavras e regras]

Como o computador entende o programa

> Programa Fonte - Linguagem de Alto Nível Ex: Python

> Compilador - Executa analise do programa

> Programa Objeto - Linguagem de Baixo Nível Ex: Assembly

> Resultado: Linguagem de máquina que o computador entende.

Análise léxica

É a primeira fase do processo de compilação. Entre suas funções está ler o código fonte e ler caractere por caractere, para fazer o processo a seguir:

Particionar > Classificar > Eliminar

Análise sintática

Sintaxe: “componente do sistema linguístico que interligam os constituintes da sentença, atribuindo-lhe uma estrutura”

Análise semântica
